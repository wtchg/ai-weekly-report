# 职业发展框架（稳定参考）

> 本文件为稳定参考框架，不随周报每周变动。周报第四部分会基于此框架给出本周的具体行动建议。

## 一、当前岗位定位

**岗位**：同花顺移动互联网事业部 · 新用户数据分析师 / 平台商业分析师

**核心能力圈**：
- 新用户留存漏斗分析与归因
- 产品运营数据支持（取数、报表、指标异动分析）
- 平台商业化相关数据洞察
- SQL + 基础 Python/R

**职业阶段**：校招入职 → 独立贡献者（IC），当前处于能力积累期

---

## 二、目标岗位与核心要求

以下岗位为当前岗位的**自然延伸方向**，基于字节、淘天、阿里国际、小红书、腾讯近 12 个月社招 JD 的共性要求提炼。

### 2.1 数据分析类（高级/资深 DA/DS）

| 维度 | 核心要求 |
|------|----------|
| 硬技能 | SQL（窗口函数、复杂子查询）、Python/R（pandas/numpy 级别即可，sklearn 加分）、AB 实验全流程、Tableau/自建看板 |
| 分析方法论 | 因果推断（DID/RDD/IV）、用户分层与 RFM、归因模型、时间序列异常检测 |
| 业务能力 | 能从"这个指标跌了"推导到"原因是 X，建议做 Y"，而不是止步于"我取了一下数" |
| 软技能 | 跨部门推动能力——能说服产品/运营接受你的分析结论并落地 |
| 薪资参考（3-5 年） | 35-60w |

**从当前岗位转型的差距**：
- AB 实验设计与评估经验（当前岗位可能接触有限）
- 因果推断方法论的系统掌握
- 从"被分配需求"到"主动发现问题"的业务感知

### 2.2 策略产品类（策略PM / 增长策略）

| 维度 | 核心要求 |
|------|----------|
| 硬技能 | SQL + 数据可视化（基础要求）、策略实验设计、推荐/搜索/推送系统基本原理 |
| 核心能力 | 策略效果的定量评估框架、ROI 测算、用户分群策略设计、收益与体验的 tradeoff 判断 |
| 业务能力 | 能把一个模糊的"提升转化"目标拆解为可执行的策略实验矩阵 |
| 软技能 | 与算法/工程的高效协作（需要理解推荐系统的基本术语和约束）、向上汇报的叙事能力 |
| 薪资参考（3-5 年） | 40-70w |

**从当前岗位转型的差距**：
- 策略设计 vs 策略评估：当前做的是"评"，需要开始练"策"
- 推荐/搜索/推送系统的技术理解
- 实验矩阵设计能力（不只是单次 AB）

### 2.3 AI 产品类（AI产品经理）

| 维度 | 核心要求 |
|------|----------|
| 硬技能 | 大模型基础原理（不需懂训练，但需懂 prompt/RAG/fine-tune 的适用场景）、SQL（需自取数据验证假设）|
| 核心能力 | AI 功能的场景判断——什么时候该用 AI、什么时候规则引擎就够了；AI 产品的评估体系建设 |
| 业务能力 | 能用数据语言和算法语言同时沟通，能设计 AI 功能的上线评估方案 |
| 软技能 | 与算法团队的高效协作、向非技术方解释 AI 能力边界 |
| 薪资参考（3-5 年） | 50-80w |

**从当前岗位转型的差距**：
- AI/LLM 技术栈的系统了解（RAG、Agent、模型评测）
- AI 产品评估方法论（LLM-as-a-Judge、幻觉率、评测集构建）
- 这个方向最前沿，变化最快，需要持续跟踪

## 三、稳定发展路径（不建议频繁变动）

```
毕业 0-2 年（当前）
  岗位：初级 DA / BA
  核心目标：独立负责一个业务模块的完整数据支持
  标志事件：能独立发现并推动至少 3 个数据驱动的业务决策
  
        ↓
        
毕业 2-4 年
  路径 A：高级数据分析师
    核心：成为某个业务线的"数据合伙人"
    能力锚：实验设计 + 因果推断 + 业务判断力
  路径 B：策略产品经理
    核心：从"分析别人做的策略"转向"自己设计策略"
    能力锚：策略设计 + 实验矩阵 + 算法协作
  路径 C：AI 产品经理
    核心：成为"懂数据、懂模型、懂业务"的桥梁角色
    能力锚：LLM 技术理解 + AI 评估体系 + 产品判断
```

**选择建议**：
- 如果喜欢深挖数据、找因果逻辑 → 路径 A
- 如果喜欢设计策略、看业务结果 → 路径 B
- 如果看好 AI 赛道、愿意承担更多不确定性 → 路径 C
- 2 年内不需要做最终选择，但要开始有意识地往一个方向积累

## 四、阶段性里程碑

| 时间节点 | 里程碑 | 产出物 |
|----------|--------|--------|
| 入职 3 个月 | 成为新用户数据的"活字典" | 能脱口而出新用户的核心漏斗数据 |
| 入职 6 个月 | 独立完成一个完整的分析项目 | 一份含实验设计、因果分析、业务建议的完整报告 |
| 入职 12 个月 | 推动一个基于你分析结论的产品决策落地 | 产品/运营直接采用你的分析建议并产生可观测的指标变化 |
| 入职 18-24 个月 | 具备跳槽/晋升的核心筹码 | 1-2 个拿得出手的分析案例 + 1 个清晰的专业方向 |
