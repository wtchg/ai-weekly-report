# 职业发展框架（稳定参考）

> 本文件为稳定参考框架，不随周报每周变动。周报的第三、四部分会基于此框架给出针对性的分析和建议。

## 一、当前岗位定位

**岗位**：同花顺移动互联网事业部 · 平台新用户数据分析师（校招入职）  
**目标行业**：纯互联网（字节跳动、小红书、阿里系、美团、腾讯等）  
**合作角色**：日常与运营团队协作最多，未来会更多参与产品项目  

**学历背景**：
- 安徽大学 金融数学 本科
- 上海财经大学 金融硕士（计量方向）
- 统计学与计量经济学基础扎实，纯数方向（随机过程、实变函数、随机分析）基础较弱

**当前技术栈**：
| 层级 | 能力 | 说明 |
|------|------|------|
| SQL | 熟练 | 日常取数主力工具 |
| Python | 不熟练 | 需要系统性提升 |
| 因果推断 | DID 掌握 | 计量背景带来的核心优势 |
| 机器学习 | 了解概念 | 知道监督/无监督学习，无实践经验 |
| 大模型 | 使用者水平 | 会用但不了解原理（RAG、fine-tune、prompt engineering） |

**当前工作内容**：
- 承接平台新用户运营的数据需求（取数、报表、指标异动分析）
- 未来核心 KPI：**扩大新用户规模 + 提升新用户转化**（开户、购买云软件产品），驱动收入增长

**职业阶段**：校招入职 → 独立贡献者（IC），当前处于能力积累期

**核心竞争策略**：计量经济学背景是大部分 DA 不具备的稀缺优势。应该围绕因果推断这条线深度积累，建立起"能做别人做不了的分析"的护城河，而不是分散精力补别人的通用技能。

## 二、目标岗位与核心要求

以下岗位为当前岗位的**自然延伸方向**，基于字节、淘天、阿里国际、小红书、美团、腾讯近 12 个月社招 JD 的共性要求提炼。

### 2.1 数据分析类（高级/资深 DA/DS）

| 维度 | 核心要求 |
|------|----------|
| 硬技能 | SQL（窗口函数、复杂子查询）、Python/R（pandas/numpy，sklearn 加分）、AB 实验全流程、Tableau/自建看板 |
| 分析方法论 | 因果推断（DID/RDD/IV/SCM）、用户分层与 RFM、归因模型、时间序列异常检测 |
| 业务能力 | 能从"这个指标跌了"推导到"原因是 X，建议做 Y"，而不是止步于"我取了一下数" |
| 软技能 | 跨部门推动能力——能说服产品/运营接受你的分析结论并落地 |
| 薪资参考（3-5 年） | 35-60w |

**用户当前差距**：
- AB 实验设计与评估经验不足
- Python 实操能力需要提升（但计量理论底子远超同类候选人）
- 需要从"承接需求"过渡到"主动发现分析命题"
- **优势**：DID 已掌握，向 RDD/IV/SCM 进阶的学习成本远低于零基础的人

### 2.2 策略产品类（策略PM / 增长策略）

| 维度 | 核心要求 |
|------|----------|
| 硬技能 | SQL + 数据可视化（基础要求）、策略实验设计、推荐/搜索/推送系统基本原理 |
| 核心能力 | 策略效果的定量评估框架、ROI 测算、用户分群策略设计、收益与体验的 tradeoff 判断 |
| 业务能力 | 能把一个模糊的"提升转化"目标拆解为可执行的策略实验矩阵 |
| 软技能 | 与算法/工程的高效协作（需要理解推荐系统的基本术语和约束）、向上汇报的叙事能力 |
| 薪资参考（3-5 年） | 40-70w |

**用户当前差距**：
- 目前做的是"策略效果的评估"，需要开始练"策略设计"
- 推荐/搜索/推送系统的技术理解
- 实验矩阵设计能力（不只是单次 AB）

### 2.3 AI 产品类（AI产品经理）

| 维度 | 核心要求 |
|------|----------|
| 硬技能 | 大模型基础原理（不需懂训练，但需懂 prompt/RAG/fine-tune 的适用场景）、SQL（需自取数据验证假设）|
| 核心能力 | AI 功能的场景判断——什么时候该用 AI、什么时候规则引擎就够了；AI 产品的评估体系建设 |
| 业务能力 | 能用数据语言和算法语言同时沟通，能设计 AI 功能的上线评估方案 |
| 软技能 | 与算法团队的高效协作、向非技术方解释 AI 能力边界 |
| 薪资参考（3-5 年） | 50-80w |

**用户当前差距**：
- AI/LLM 技术栈的系统了解（RAG、Agent、模型评测）
- 但用户的计量背景在 AI 评估（LLM-as-a-Judge 的统计验证、评测集设计的抽样方法论）中有独特优势

## 三、稳定发展路径（不建议频繁变动）

```
毕业 0-2 年（当前）
  岗位：初级 DA / BA（同花顺平台新用户方向）
  核心任务：
    1. 成为新用户数据的"活字典"
    2. 把 DID 从"论文用过"变成"业务中能用"
    3. 系统性提升 Python（pandas → statsmodels → sklearn）
    4. 至少独立推动 3 个基于数据的产品/运营决策
  标志事件：能用 DID/RDD 独立完成一次归因分析，且结论被产品/运营采纳
  
        ↓
        
毕业 2-4 年
  路径 A：高级数据分析师（互联网大厂）
    核心壁垒：因果推断方法论深度 + 实验设计经验 + 一个成熟业务线的完整分析案例集
    能力锚：DID → RDD → IV → SCM 的完整因果推断工具箱 + AB 实验全流程
    
  路径 B：策略产品经理（互联网大厂）
    核心壁垒："数据分析+策略设计+实验评估"三合一
    能力锚：在路径 A 的基础上 + 策略设计与 ROI 测算 + 推荐系统理解
    
  路径 C：AI 产品经理（互联网大厂）
    核心壁垒：LLM 评估体系搭建 + 数据驱动决策
    能力锚：因果推断 + AI 评估方法论 + 产品判断力
```

**选择建议**：
- 如果喜欢深挖数据、找因果逻辑 → 路径 A（用户的计量背景有天然优势）
- 如果喜欢设计策略、看业务结果 → 路径 B（路径 A 积累 1-2 年后自然转型）
- 如果看好 AI 赛道 → 路径 C（核心壁垒是"懂评估的人很少"，计量背景在这里很稀缺）
- **强烈建议 2 年内先走路径 A 的能力积累，不急着选死一个方向。** 因果推断 + 实验设计 + Python 自动化是三条路径共同的基石。

## 四、阶段性里程碑

| 时间节点 | 里程碑 | 产出物 |
|----------|--------|--------|
| 入职 3 个月 | 成为新用户数据的"活字典" | 能脱口而出新用户的核心漏斗数据 |
| 入职 6 个月 | 独立完成一个完整的分析项目 | 一份含实验设计、因果分析、业务建议的完整报告 |
| 入职 12 个月 | 推动一个基于你分析结论的产品决策落地 | 产品/运营直接采用你的分析建议并产生可观测的指标变化 |
| 入职 18-24 个月 | 具备跳槽/晋升的核心筹码 | 1-2 个拿得出手的分析案例 + 1 个清晰的专业方向 |
